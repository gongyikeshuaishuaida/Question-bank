# PDF 试卷提取方案

当你提供试卷 PDF 原稿时，目标是提取信息技术部分，生成符合 `题库_模板.md` 的 Obsidian 题目笔记。

---

## 工作流程

### 1. 提供试卷

将试卷 PDF 文件放入 Obsidian vault，并说明试卷信息：

- 时间：如 `202604`
- 组织：如 `金华十校`
- 类型：仅限 `期中`、`期末`、`月考`、`期初`
- 提取范围：只提取信息技术部分

### 2. 执行提取

```
试卷 PDF
    ↓
① 读取 PDF 内容（文字 + 图片）
    ↓
② 识别信息技术部分和题目边界
    ↓
③ 保留原文提取每道题
    ↓
④ 提取题目局部图片 → 保存到 attachments/
    ↓
⑤ 在 试卷/ 中保存 PDF 原稿，命名为 时间-组织-类型.pdf
    ↓
⑥ 生成符合模板的 md 文件
    ↓
⑦ 补全答案和解析
    ↓
⑧ 按主知识点放入分类目录
```

### 3. 输出结果

- 试卷 PDF 原稿存放在 `试卷/` 文件夹，命名为 `时间-组织-类型.pdf`
- 每道题一个 Markdown 文件，文件名必须与 `id` 保持一致，即 `id.md`
- 每道题必须有稳定 `id`，格式为 `YYYYMM-组织-类型-题号`
- `来源` 字段链接到 `试卷/` 中的 PDF 原稿
- 图片保存到 `attachments/`
- frontmatter 使用中文字段
- 统一添加 `状态/未练习`、`知识点/...` 标签，不添加 `题库/信息技术`、`题型/...` 等可由字段推导的冗余标签
- 含图片题目默认 `图片核验: 待核验`，无图片题目填写 `图片核验: 无图片`
- 每道题必须写出答案和解析

## 图片处理方案

### 图片提取

| 图片类型 | 处理方式 |
|---|---|
| 题目配图 | 提取为 PNG，嵌入题目内容区 |
| 选项图片 | 提取为 PNG，嵌入选项区 |
| 代码截图 | 优先 OCR 转为代码块；无法可靠识别时保留图片 |
| 表格图片 | 优先识别为 Markdown 表格；复杂表格保留图片 |
| 流程图、树、图表 | 只截取题目所需局部 |

### 图片命名规则

```
attachments/
├── 金华十校_01_图1.png
├── 金华十校_07_图1.png
└── 金华十校_15_图1.png
```

### 图片引用方式

```markdown
![[attachments/金华十校_15_图1.png]]
```

### 图片核验规则

- 图片必须是题目对应的局部截图
- 不允许把整页试卷截图保存为题目图片
- 图片下方不添加 `> 上图：...` 这类说明文字
- 自动提取后的图片先标记为 `待核验`
- 人工确认裁切范围、清晰度、题目对应关系后，改为 `已通过`
- 没有图片的题目填写 `无图片`

## 题目识别规则

| 特征 | 说明 |
|---|---|
| 题号 | `1.`、`2.`、`第1题`、`（1）` 等格式 |
| 题型标记 | `选择题`、`填空题` |
| 选项标记 | `A.`、`B.`、`C.`、`D.` |
| 分值标记 | 如 `10分`、`共9分` |
| 材料题 | 材料原文整体保留，相关小题放在同一题目文件中 |

### 材料题处理

- 遇到 `阅读材料，完成选择题1-3` 这类题组时，每个 md 文件都包含完整材料和对应题组的全部小题
- 同一材料题组按题号复制多份，例如第1~3题生成 3 个文件
- 3 个文件的 `id`、文件名、标题题号分别为 `01`、`02`、`03`
- 每份文件按对应题目的主要知识点分类，放入对应文件夹

---

## 答案和解析

- 答案和解析由提取者在生成题目时完成
- 不留空，不写 `待补充`
- 选择题写明正确选项
- 填空题逐空列出答案
- 解析说明关键判断依据、排除理由或代码执行过程
- 若试卷原稿不含参考答案，则根据题目内容自行推导答案，并在解析中体现依据

---

## 手动确认内容

- [ ] 知识点：确认大知识点与细分知识点
- [ ] 难度：简单/中等/困难
- [ ] 图片核验：确认图片为题目局部截图
- [ ] 复习字段：维护 `完成次数`、`正确率`、`状态`、`错题原因`

## 注意事项

> [!warning] PDF 格式要求
> - 扫描版 PDF 需要先 OCR 识别
> - 图片质量越高，提取效果越好
> - 复杂排版可能需要手动调整

> [!tip] 提高识别准确率
> - 题号格式统一
> - 题目之间有明确分隔
> - 选项格式统一
> - 材料题和小题边界清晰
