# 题库管理规则

## 一、文件夹结构

```
Obsidian Vault/
├── 01数据与编码/     # 信息数据知识智慧、数据采集与编码、进制转换、数字化、数据管理与大数据
├── 02算法/          # 算法定义、三要素、五个特征、表达方式、流程图（代码题不要放入）
├── 03python基础/    # Python语法、数据类型、运算符、字符串/列表/字典、分支循环、函数；试卷13、14题中的一般 Python 代码题
├── 04数据处理/      # Excel/pandas/DataFrame/read_excel/groupby 等数据表处理、可视化
├── 06人工智能/      # AI分类、应用、发展、概念、主义
├── 07信息系统/      # 系统分类、组成、搭建
├── 08信息安全/      # 安全概念、加密、访问控制、身份认证、密钥、防火墙、病毒、漏洞
├── 09数组/          # 数组相关题目
├── 10队列/          # 队列相关题目
├── 11栈/            # 栈相关题目
├── 12链表/          # 链表相关题目
├── 13树/            # 树相关题目
├── 14查找与排序/    # 顺序查找、二分查找、冒泡/选择/插入/桶/计数排序
├── 15迭代与递归/    # 迭代与递归，算法复杂度
├── 试卷/            # 每次处理的 PDF 原稿文件，命名为 时间-组织-类型.pdf
├── attachments/     # 图片文件夹
└── 题库索引.md      # 索引文件
```

## 二、文件命名规则

### 2.1 题目文件命名

**格式**：`时间-组织-类型-题号.md`

**示例**：
- `202604-湖衢丽-期中-08.md`
- `202504-金华十校-月考-15.md`

**说明**：
- 时间：6位数字，如 `202604` 表示2026年4月
- 组织：如 `湖衢丽`、`金华十校`、`浙江` 等
- 类型：`期中`、`期末`、`月考`、`期初`
- 题号：**2位数字**，如 `01`、`15`

**重要**：题号必须统一为**两位数**格式
- ✅ 正确：`202604-湖衢丽-期中-01.md`、`202604-湖衢丽-期中-15.md`
- ❌ 错误：`202604-湖衢丽-期中-001.md`、`202604-湖衢丽-期中-1.md`

题目文件名必须和 frontmatter 中的 `id` 保持一致，即 `id.md`。

### 2.2 试卷文件命名

- 试卷统一存放在 `试卷/` 文件夹
- 试卷文件是 PDF 原稿文件
- 命名格式：`时间-组织-类型.pdf`
- 示例：`202604-金华十校-期中.pdf`
- 同一试卷下的题目 `id` 使用该试卷名作为前缀，例如 `202604-金华十校-期中-01`
- 只有收到或处理 PDF 原稿时才添加试卷文件，不额外创建试卷索引 md 文件

## 四、格式规范

### 4.1 标题规范
- ✅ 使用 `# 题目 题号` 作为题目标题，如 `# 题目 01`
- ❌ 不要使用 `# 题目内容`、`## 问题`、`## 选项` 等标题
- 选项直接作为题目内容的一部分，不单独分节
- 选择题每个题干末尾必须统一写 `（   ）`，不要省略括号，也不要使用 `（▲）`；若原题题干末尾有句号、问号等标点，统一去掉题干末尾标点后保留 `（   ）`

### 4.2 Frontmatter 标签
- `tags` 只保留状态类和知识点类标签，例如 `状态/未练习`、`知识点/03python基础`
- 不写可由字段推导的冗余标签，例如 `题库/信息技术`、`题型/选择题`

### 4.3 代码格式
- **代码块**：使用 ` ```python ... ``` ` 格式
- **行内代码**：Python表达式用反引号包裹，如 `` `df["列名"]` ``、`` `a[i]` ``
- 特别注意：包含 `==`、`[]`、`()` 的表达式需要用反引号

### 4.4 图片引用
- 图片存放在 `attachments/` 文件夹
- 引用格式：`![[attachments/图片名.png]]`
- 图片命名：`试卷时间试卷简称_题号_序号.png`，如 `202604湖衢丽_p02_img01.png`

### 4.5 图片核验
- 每个含图片的题目必须填写 frontmatter 的 `图片核验` 字段
- 可选值：`无图片`、`待核验`、`已通过`
- `已通过` 表示图片确认为题目对应的局部截图，例如统计图、流程图、树结构或选项图
- 不允许把整页试卷截图作为题目图片保存
- PDF 提取后，凡是自动截取的图片先标记为 `待核验`，人工确认后再改为 `已通过`
- 图片下方不要添加 `> 上图：...` 这类说明文字

### 4.6 答案和解析
- 每道题必须包含 `## 答案` 和 `## 解析`
- 答案和解析由提取者在生成题目时完成，不留空，不写 `待补充`
- 选择题答案格式：`**正确答案：** A`
- 多选题答案格式：`**正确答案：** A、C`
- 填空题答案格式：逐空列出，例如 `① ...；② ...`
- 解析要说明关键判断依据、排除理由或代码执行过程
- 解析不需要写成长篇讲义，但必须足够支持复习时理解为什么这样作答

## 五、知识点分类规则

### 5.1 分类依据
根据题目涉及的主要知识点放入对应文件夹

### 5.2 分类优先级
- `02算法` 只放算法概念题和流程图题，例如算法三要素、算法五个特征、流程图执行。只要是 Python 代码题，即使涉及贪心、滑动窗口、状态模拟等算法思想，也不要放入 `02算法`。
- 试卷第 `13`、`14` 题中的一般 Python 代码题优先放入 `03python基础`。如果涉及数组、排序等内容，在 frontmatter 的 `知识点` 中补充 `09数组`、`14查找与排序` 等次级知识点。
- `04数据处理` 只放明确使用 pandas、Excel 数据表、DataFrame、`read_excel`、`groupby`、统计图表绘制等数据表处理的题。普通列表、数组或传感器数据模拟处理不放入 `04数据处理`。
- 试卷第 `15` 题一般按数据结构归入 `09数组`、`10队列`、`11栈`、`12链表`。优先判断是否属于队列、栈、链表；若都不属于，再放入 `09数组`。若同时涉及排序，在 frontmatter 的 `知识点` 中补充 `14查找与排序`。
- 综合题同时涉及 `07信息系统` 和 `04数据处理` 时，按小问数量和主要考查内容决定主目录；frontmatter 的 `知识点` 中同时保留 `07信息系统` 和 `04数据处理`，保证两类检索都能命中。

### 5.3 跨知识点题目
- 非材料题涉及多个知识点时，主文件只放入一个最主要的知识点文件夹
- 其他相关知识点写入 frontmatter 的 `知识点` 字段
- 例如：一道题主要考查栈，同时涉及队列，则文件放入 `11栈`，并在 `知识点` 中同时写入 `11栈`、`10队列`

### 5.4 知识点标签
在 frontmatter 的 `知识点` 字段中：
- 第一项写大知识点（如 `01数据与编码`）
- 后续可写细分知识点

### 5.5 题目唯一 ID
- 每道题必须填写稳定的 `id`
- 格式：`YYYYMM-组织-类型-题号`
- 示例：`202604-金华十校-期中-01`
- 题目文件名必须与 `id` 一致，即 `202604-金华十校-期中-01.md`
- 题目移动到其他知识点文件夹时，`id` 和文件名都不变

### 5.6 复习字段
- `完成次数`：累计练习次数，初始为 `0`
- `正确率`：当前统计正确率，使用 0~1 的小数记录，例如 `1`、`0.92`、`0.75`；未练习时留空
- `状态`：可选 `未练习`、`已完成`、`需复习`、`错题`
- `错题原因`：记录错误类型或原因，例如 `概念混淆`、`代码跟踪错误`、`审题错误`
- 批量导入练习统计时，选择题可按外部统计表同步 `完成次数`、`正确率` 和 `状态`

## 六、试卷提取规则

### 6.1 提取范围
- ✅ **只提取信息技术部分**（通常是试卷前半部分）
- ❌ 不提取通用技术部分

### 6.2 提取要求
- 题目内容**原封不动**，不做概括或改写
- 材料题要整体提取，并按题号复制为多个 md 文件
- 例如：`阅读材料，完成选择题1-3`，则每个 md 文件中都应包含完整材料和选择题1-3
- 材料题中的“对应题组”指该材料下面的**所有相关选择题**，不是只写当前题号的小题。例如 `202604-金华十校-期中-01` 对应材料覆盖第 1-6 题，则 `202604-金华十校-期中-01` 到 `202604-金华十校-期中-06` 每个 md 正文都必须包含完整材料和第 1-6 题全部题干、选项；只允许各文件的 `id`、标题题号、分类、答案和解析焦点不同。
- 上例需要复制 3 份，`id`、文件名和标题题号分别为 `01`、`02`、`03`
- 复制后的每份文件按对应题目的主要知识点分类，放入对应文件夹
- 保留原始题干、代码、选项
- 图片需要提取试卷中对应的内容并保存，注意一定要是对应内容，不要整张页面截图进来
- 例如：题干中有统计图，你就只截取统计图进来，题干中有树，你就只截取树进来

### 6.3 提取后处理
1. 按知识点分类存放到对应文件夹
2. 材料题按题号复制多份，并分别分类
3. 跨知识点题目补充多个 `知识点` 字段
4. 为每道题写出答案和解析
5. 将 PDF 原稿放入 `试卷/`
6. 更新题库索引

---

## 七、注意事项

1. **不要概括题目**：保持试卷原文
2. **代码格式正确**：避免与Markdown语法冲突
3. **图片路径正确**：使用相对路径
4. **知识点分类准确**：便于后续查询
5. **答案解析完整**：生成题目时必须同步完成答案和解析
