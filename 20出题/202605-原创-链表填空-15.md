# 题目 15

某学校机房只有一台共享打印机。学生和教师提交打印任务后，系统会将任务加入待打印链表中。每个任务节点用列表 `[任务编号, 提交时间, 页数, 紧急程度, next]` 表示，其中提交时间已转换为当天 0 点起的分钟数，紧急程度数值越小表示越紧急，`next` 存储下一个节点在列表 `jobs` 中的下标，`-1` 表示空指针。

待打印链表按如下规则排序：先按紧急程度升序排列；紧急程度相同，再按提交时间升序排列；若提交时间也相同，则按任务编号升序排列。打印机每分钟打印 1 页。当打印机空闲时，系统从待打印链表表头取出一个任务打印。

**（1）** 已知某时刻 `jobs` 和链表头指针 `head` 如下：

```python

    [1,486,6,1,3],

    [3,480,10,0,4],
    [4,488,5,1,1],
    [5,484,7,0,0]
]
head = 2
```

从表头开始，第 3 个待打印任务的任务编号是 ______。

**（2）** 函数 `before(a,b)` 用于判断节点 `a` 是否应排在节点 `b` 前面。若当前 `jobs` 和 `head` 如下：

## 答案

**正确答案：** （1）1；（2）4；（3）① `before(jobs[p][4], i)`；② `jobs[p][4]=i`；③ `jobs_raw[i][1]`

---

## 解析

（1）从 `head=2` 开始沿指针遍历，链表顺序为下标 `2 -> 4 -> 0 -> 3 -> 1`，对应任务编号为 `3 -> 5 -> 1 -> 4 -> 2`，第 3 个待打印任务编号为 1。

（2）下标为 5 的任务为 `[6,488,4,1,-1]`。原链表顺序为下标 `2 -> 0 -> 4 -> 3 -> 1`，其中紧急程度为 1 的节点按提交时间依次为下标 0、4、3。新任务提交时间为 488，应插入下标 0 与下标 4 之间，因此插入后 `jobs[5][4]=4`。

（3）`before(a,b)` 表示节点 `a` 应排在节点 `b` 前面。插入新节点 `i` 时，若后继节点 `jobs[p][4]` 仍应排在 `i` 前面，就继续后移，所以①为 `before(jobs[p][4], i)`；找到插入位置后，先令新节点指向原后继，再令当前节点 `p` 指向新节点，所以②为 `jobs[p][4]=i`。调度时若链表为空但仍有未提交任务，打印机需要等待到下一项任务提交时间，因此③为 `jobs_raw[i][1]`。
